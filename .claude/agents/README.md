# Slidys Development Subagent Workflow

このドキュメントでは、Slidysプロジェクトでのサブエージェントを活用した開発ワークフローを説明します。

## プロジェクト概要

Slidysは、SwiftUIベースのプレゼンテーション（スライド）アプリです。SlideKitを使用して、カンファレンス発表用のスライドを作成・表示します。

## ワークフロー概要

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    メインセッション（オーケストレーター）                  │
│                                                                         │
│  ┌───────────────────────────────────────────────────────────────────┐  │
│  │ Phase 1: 調査・計画                                                │  │
│  │   Task(ios-planner)                                               │  │
│  │     └── 実装計画作成 → 「Implementation Plan」を返却               │  │
│  └───────────────────────────────────────────────────────────────────┘  │
│                                    ↓                                    │
│  ┌───────────────────────────────────────────────────────────────────┐  │
│  │ Phase 2: 実装                                                      │  │
│  │   Task(ios-implementer, prompt="Implementation Plan")             │  │
│  │     └── 実装レポート → 「Implementation Report」を返却             │  │
│  └───────────────────────────────────────────────────────────────────┘  │
│                                    ↓                                    │
│  ┌───────────────────────────────────────────────────────────────────┐  │
│  │ Phase 2.5: レビュー（チェック＋修正）                               │  │
│  │   Task(ios-code-reviewer)       → コーディング規約チェック＋修正   │  │
│  │   Task(ios-unit-test-reviewer)  → テストコードチェック＋修正       │  │
│  └───────────────────────────────────────────────────────────────────┘  │
│                                    ↓                                    │
│  ┌───────────────────────────────────────────────────────────────────┐  │
│  │ Phase 3: 検証（並列実行）                                          │  │
│  │   Task(ios-build-checker)   → ビルド結果 + Fix Plan               │  │
│  │   Task(ios-test-checker)    → テスト結果 + Fix Plan (optional)    │  │
│  └───────────────────────────────────────────────────────────────────┘  │
│                                    ↓                                    │
│  ┌───────────────────────────────────────────────────────────────────┐  │
│  │ Phase 4: 修正（エラーがある場合のみ）                               │  │
│  │   Task(ios-fixer, prompt="Consolidated Fix Plan")                 │  │
│  │     └── Phase 3に戻る（再検証）                                    │  │
│  └───────────────────────────────────────────────────────────────────┘  │
│                                    ↓                                    │
│  ┌───────────────────────────────────────────────────────────────────┐  │
│  │ Phase 5: 完了報告                                                  │  │
│  └───────────────────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────────────────┘
```

## サブエージェント一覧

### 調査・計画
| Agent | 用途 |
|-------|------|
| `ios-planner` | コードベース調査・実装計画作成 |

### 実装
| Agent | 用途 |
|-------|------|
| `ios-implementer` | 計画に基づく実装実行 |
| `ios-fixer` | チェッカーからの修正実行 |

### レビュー（チェック＋修正）
| Agent | 用途 | 実行タイミング |
|-------|------|---------------|
| `ios-code-reviewer` | コーディング規約チェック＋修正 | 常に実行 |
| `ios-unit-test-reviewer` | テストコードチェック＋修正 | テストファイル変更時 |

### 検証（チェックのみ）
| Agent | 用途 |
|-------|------|
| `ios-build-checker` | ビルド実行・エラーレポート |
| `ios-test-checker` | UnitTest実行・失敗レポート |

## 注意事項

1. **並列実行の活用**: 検証フェーズはチェッカーを並列実行して効率化
2. **ループの上限**: 修正→再検証のループは最大3回まで（無限ループ防止）
3. **手動介入**: 自動修正できない場合はユーザーに判断を仰ぐ
4. **コンテキスト節約**: 詳細ログはサブエージェント内に留め、メインには要約のみ返却
5. **ビルドコマンド**: このプロジェクトではSwift Package Managerまたはxcodebuildを直接使用（カスタムスクリプトなし）
